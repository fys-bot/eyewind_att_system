# 跨月规则调试日志说明

## 问题描述
每个月的第一天没有去计算前一个月最后一天的下班实际打卡时间。

## 已实施的修复

### 1. 数据加载范围扩展 (AttendanceDashboardPage.tsx)
- **位置**: `loadAllData` 方法
- **修复内容**: 
  - 计算回溯天数：默认5天，如果本月第一天是周一或周日则回溯7天
  - 确保能获取到上月最后几天的打卡数据
  - 数据范围：从 `本月1日 - 回溯天数` 到 `下月1日`

### 2. 跨月规则实现 (useAttendanceStats.ts)
- **位置**: `useAttendanceStats` hook
- **实现逻辑**:
  1. 判断当前日期是否是本月第一个工作日
  2. 如果是，从 `user.punchData` 中查找上个月的所有下班打卡
  3. 取最晚的一条作为 `previousMonthCheckoutTime`
  4. 传递给规则引擎计算迟到分钟数

### 3. 规则引擎支持 (AttendanceRuleEngine.ts)
- **位置**: `calculateLateMinutes` 方法
- **优先级**:
  1. 跨月规则（本月第一个工作日）
  2. 跨周规则（周一）
  3. 跨天规则（普通工作日）
  4. 默认规则（标准工作开始时间）

## 验证步骤

1. **清除缓存**: 在浏览器开发者工具中清除 IndexedDB 缓存
2. **选择月份**: 切换到需要测试的月份（例如 2026-02）
3. **查看控制台**: 检查以下日志输出
   - 数据加载范围是否包含上月数据
   - 冯永森和徐怡的打卡数据是否包含上月记录
   - 是否检测到本月第一个工作日
   - 是否找到上月最后下班打卡时间
   - 是否应用了跨月规则

## 预期结果

对于每月第一个工作日：
- 如果前一个月最后一个工作日下班时间 >= 20:30，次日 09:31 前打卡不算迟到
- 如果前一个月最后一个工作日下班时间 >= 24:00，次日 13:31 前打卡不算迟到
- 如果前一个月最后一个工作日下班时间 < 18:30，使用默认规则（09:00 算迟到）

## 可能的问题

### 问题1: 上月数据未加载
**症状**: 日志显示 "上月记录数: 0"
**原因**: 
- 数据加载范围不够（回溯天数不足）
- API 未返回上月数据
- 缓存中的数据不包含上月数据

**解决方案**: 
- 增加回溯天数（修改 `daysToGoBack` 变量）
- 清除缓存后重新加载
- 检查 API 返回的数据范围

### 问题2: 跨月规则未触发
**症状**: 日志未显示 "检测到本月第一个工作日"
**原因**:
- 当前日期不是本月第一个工作日
- 节假日配置错误导致工作日判断不准确

**解决方案**:
- 检查节假日配置
- 验证工作日判断逻辑

### 问题3: 找不到上月下班打卡
**症状**: 日志显示 "上月下班打卡总数: 0"
**原因**:
- 员工上月确实没有下班打卡记录
- 数据过滤逻辑有误

**解决方案**:
- 检查员工上月的实际打卡情况
- 验证 `parseWorkDate` 函数的日期解析逻辑
- 检查 `checkType` 和 `timeResult` 的过滤条件
