# 跨月/跨周规则数据获取修复

## 问题描述

用户报告：所有员工都显示"上月无下班打卡记录"，但这不可能是真的。

## 根本原因

在 `AttendanceDashboardPage.tsx` 中，数据获取的日期范围设置为：
- `fromDate`: 当前月份的第2天（例如 `2026-02-02`）
- `toDate`: 下个月的第1天（例如 `2026-03-01`）

这意味着 `fetchCompanyData` 只获取当前月份的打卡数据，**不包含上个月的任何数据**。

### 影响的场景

1. **跨月规则失效**
   - 当查找"上月最后一个工作日的下班打卡时间"时
   - `user.punchData` 中没有上个月的记录
   - 所有员工都显示"上月无下班打卡记录"
   - 默认使用 18:30 作为上月下班时间（不准确）

2. **跨周规则失效（边界情况）**
   - 如果本月第一天是周一，上周末（周五/周六/周日）在上个月
   - `userAttendance` 中没有上周末的数据
   - 跨周规则无法正确应用

## 修复方案

### 智能回溯数据获取

**文件**: `components/attendance/dashboard/AttendanceDashboardPage.tsx`

**修改前**:
```typescript
const fromDate = `${globalMonth}-02`;
```

**修改后**:
```typescript
// 获取本月第一天是星期几
const [y, m] = globalMonth.split('-').map(Number);
const firstDayOfMonth = new Date(y, m - 1, 1);
const firstDayOfWeek = firstDayOfMonth.getDay();

// 计算需要回溯多少天
let daysToGoBack = 1; // 默认至少获取上月最后一天

if (firstDayOfWeek === 1) {
  // 如果本月第一天是周一，需要获取上周五/六/日（回溯3天）
  daysToGoBack = 3;
} else if (firstDayOfWeek === 0) {
  // 如果本月第一天是周日，需要获取上周五/六（回溯2天）
  daysToGoBack = 2;
}

// 计算起始日期
const startDate = new Date(y, m - 1, 1 - daysToGoBack);
const fromDate = `${startDate.getFullYear()}-${String(startDate.getMonth() + 1).padStart(2, '0')}-${String(startDate.getDate()).padStart(2, '0')}`;
```

**效果**:
- 根据本月第一天是星期几，智能决定回溯天数
- 确保跨月和跨周规则都能找到所需的数据
- 最小化数据传输（只获取必要的天数）

### 回溯逻辑说明

| 本月第一天 | 回溯天数 | 获取的日期范围 | 原因 |
|----------|---------|--------------|------|
| 周一 | 3天 | 上月29/30/31日 | 需要上周五/六/日数据（跨周规则） |
| 周日 | 2天 | 上月30/31日 | 需要上周五/六数据（跨周规则） |
| 周二-周六 | 1天 | 上月31日 | 只需要上月最后一天（跨月规则） |

## 测试验证

### 测试场景 1：跨月规则

**条件**: 查看2月份考勤，2月3日（周一）是第一个工作日

**预期**:
- 数据范围：`2026-01-31` 到 `2026-03-01`
- 能找到1月31日的下班打卡时间
- 如果1月31日加班到20:30+，2月3日可9:30上班

**验证日志**:
```
[跨月规则] 张三 - 找到上月最后下班打卡: 2026-01-31T22:30:00.000Z
[AttendanceRuleEngine] 应用跨月规则: 22:30→09:30, 迟到0分钟
```

### 测试场景 2：跨周规则（本月第一天是周一）

**条件**: 查看3月份考勤，3月3日（周一）是第一天

**预期**:
- 数据范围：`2026-02-28` 到 `2026-04-01`（回溯3天）
- 能找到2月28日（周五）、3月1日（周六）、3月2日（周日）的打卡
- 如果周末有加班，周一可享受晚到优惠

**验证日志**:
```
[AttendanceRuleEngine] 检查跨周打卡规则: 2026-03-03, 前一时段下班: 2026-03-02T23:00:00.000Z
[AttendanceRuleEngine] 应用跨周规则: 23:00→09:30, 迟到0分钟
```

### 测试场景 3：普通情况（本月第一天是周三）

**条件**: 查看4月份考勤，4月1日（周三）是第一天

**预期**:
- 数据范围：`2026-03-31` 到 `2026-05-01`（回溯1天）
- 只获取必要的数据，减少传输量

## 影响范围

### 正面影响

1. **跨月规则正确应用**
   - 上月加班的员工，本月第一天可以享受晚到优惠
   - 迟到计算更准确

2. **跨周规则正确应用（边界情况）**
   - 本月第一天是周一时，能正确找到上周末的打卡数据
   - 周末加班的员工，周一可以享受晚到优惠

3. **数据完整性**
   - `punchData` 包含跨月/跨周所需的所有数据
   - 不再依赖默认值（18:30）

4. **智能优化**
   - 根据实际需要决定回溯天数
   - 最小化不必要的数据传输

### 潜在影响

1. **数据量增加**
   - 每次查询会多获取1-3天的数据
   - 影响很小（最多3天的数据）

2. **缓存键变化**
   - 缓存键包含 `fromDate`，修改后会导致旧缓存失效
   - 用户首次访问时需要重新获取数据

- `components/attendance/dashboard/useAttendanceStats.ts` - 跨月规则逻辑（未修改）
- `components/attendance/AttendanceRuleEngine.ts` - 规则引擎（未修改）

## 后续优化建议

1. **缓存优化**
   - 分离上月数据和当月数据的缓存
   - 避免因上月数据变化导致整个缓存失效

2. **预加载策略**
   - 在查看当月数据时，预加载下个月第一周的数据
   - 提升切换月份时的响应速度

3. **数据压缩**
   - 对于历史月份，只获取必要的字段
   - 减少数据传输量

## 总结

通过智能计算需要回溯的天数，从"当前月第2天"改为"上个月最后1-3天"，解决了跨月和跨周规则无法找到上月打卡数据的问题。这个修复：
- ✅ 解决了跨月规则数据缺失问题
- ✅ 解决了跨周规则边界情况问题
- ✅ 最小化了数据传输量
- ✅ 保持了代码的简洁性
