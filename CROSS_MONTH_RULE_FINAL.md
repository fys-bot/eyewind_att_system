# 跨月规则最终版本说明

## 版本日期
2026-02-25

## 功能描述
实现了完整的跨月打卡规则，支持根据上月最后一个工作日的下班打卡时间，动态调整本月第一个工作日的迟到基准时间。

## 核心逻辑

### 1. 数据加载 (AttendanceDashboardPage.tsx)
- 自动计算回溯天数：
  - 默认回溯 5 天
  - 如果本月第一天是周一或周日，回溯 7 天
- 数据范围：从 `本月1日 - 回溯天数` 到 `下月1日`
- 确保能获取到上月最后几天的打卡数据

### 2. 跨月规则检测 (useAttendanceStats.ts)
- 判断当前日期是否是本月第一个工作日
- 从 `user.punchData` 中查找上个月的所有下班打卡记录
- 取最晚的一条作为 `previousMonthCheckoutTime`
- 如果上月没有下班打卡记录，默认使用 18:30

### 3. 规则引擎计算 (AttendanceRuleEngine.ts)

#### 优先级顺序
1. **跨月规则**（本月第一个工作日）
2. **跨周规则**（周一）
3. **跨天规则**（普通工作日）
4. **默认规则**（标准工作开始时间 09:00）

#### 特殊处理：次日凌晨下班
如果上月最后下班时间是次日凌晨（00:00-06:00），系统会：
- 识别为"超过24:00"的情况
- 直接应用 24:00 规则（13:31 算迟到）
- 不再比较实际的下班时间分钟数

#### 规则匹配逻辑
```
上月最后下班时间          本月第一天迟到基准
< 18:30                → 09:01
18:30 - 20:29          → 09:01
20:30 - 23:59          → 09:31
>= 24:00 (次日凌晨)     → 13:31
```

## 关键代码位置

### AttendanceDashboardPage.tsx
- **方法**: `loadAllData`
- **行数**: ~280-320
- **功能**: 计算回溯天数，加载包含上月数据的打卡记录

### useAttendanceStats.ts
- **方法**: `useAttendanceStats` hook
- **行数**: ~750-850
- **功能**: 检测本月第一个工作日，查找上月最后下班打卡

### AttendanceRuleEngine.ts
- **方法**: `calculateLateMinutes`
- **行数**: ~80-180
- **功能**: 根据优先级应用跨月/跨周/跨天规则

- **方法**: `applyLateRulesForCrossDay`
- **行数**: ~240-360
- **功能**: 
  - 检测次日凌晨下班（00:00-06:00）
  - 直接应用 24:00 规则
  - 匹配其他规则阈值

## 测试场景

### 场景1: 上月最后下班 20:30-23:59
- **上月最后下班**: 2026-01-31 22:30
- **本月第一天**: 2026-02-02（周一，第一个工作日）
- **迟到基准**: 09:31
- **预期**: 09:31 前打卡不算迟到

### 场景2: 上月最后下班次日凌晨
- **上月最后下班**: 2026-02-01 00:00:22（即 2026-01-31 24:00:22）
- **本月第一天**: 2026-02-02（周一，第一个工作日）
- **迟到基准**: 13:31
- **预期**: 13:31 前打卡不算迟到

### 场景3: 上月最后下班 < 18:30
- **上月最后下班**: 2026-01-31 18:00
- **本月第一天**: 2026-02-02
- **迟到基准**: 09:01
- **预期**: 09:01 前打卡不算迟到

## 已知限制

1. **回溯天数限制**: 最多回溯 7 天，如果上月最后 7 天都是节假日，可能无法获取到上月数据
2. **时区处理**: 假设所有时间都是本地时区（北京时间）
3. **数据依赖**: 依赖 `user.punchData` 包含上月数据，如果 API 未返回上月数据，规则无法生效

### 常见问题

**Q: 为什么跨月规则没有生效？**
A: 检查以下几点：
- 是否是本月第一个工作日
- 上月数据是否已加载（查看 punchData）
- 规则配置是否正确
- crossDayCheckout.enabled 是否为 true

**Q: 如何验证规则是否正确？**
A: 
1. 临时启用调试日志
2. 查看控制台输出
3. 验证数据加载范围
4. 验证规则匹配过程

**Q: 次日凌晨下班的判断标准是什么？**
A: 下班时间的小时数在 0-6 之间（即 00:00-06:00）
